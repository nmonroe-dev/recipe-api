Recipe API Documentation

Welcome to the Recipe API documentation. This guide will help you understand how to use the API to access and manage recipes. You can perform operations such as retrieving, adding, updating, and deleting recipes.

Base URL

All API requests should be made to the following base URL:

http://your-api-url.com/api

This base URL serves as the root endpoint for all API operations.

Endpoints and Methods

GET /recipes

Description: Retrieves a list of all recipes available in the database.

Example Request:

GET http://your-api-url.com/api/recipes

Response: Returns a JSON array of recipes.

GET /recipe/{id}

Description: Retrieves details of a specific recipe by its unique ID.

Example Request:

GET http://your-api-url.com/api/recipe/chicken-and-vegetable-stir-fry

Response: Returns a JSON object containing the recipe details.

POST /recipe

Description: Adds a new recipe to the database.

Example Request Body:

{
  "name": "New Recipe",
  "ingredients": ["ingredient1", "ingredient2"],
  "instructions": "Cooking instructions.",
  "img": "URL_to_image"
}

Response: Returns the newly created recipe with its ID.

PUT /recipe/update/{id}

Description: Updates an existing recipe specified by its ID.

Example Request Body:

{
  "ingredients": ["ingredient1", "ingredient2"],
  "instructions": "Updated cooking instructions.",
  "img": "URL_to_image"
}

Response: Returns the updated recipe details.

DELETE /recipe/delete/{id}

Description: Deletes a specific recipe from the database using its ID.

Example Request:

DELETE http://your-api-url.com/api/recipe/delete/chicken-and-vegetable-stir-fry

Response: Returns a confirmation message of deletion.

Error Handling

When interacting with the Recipe API, you might encounter errors. Below are common error responses and their meanings:

400 Bad Request

Description: The request could not be understood or was missing required parameters.

Possible Fix: Check the request syntax and ensure all required parameters are included.

404 Not Found

Description: The requested resource could not be found.

Possible Fix: Verify that the URL is correct and that the resource exists.

500 Internal Server Error

Description: An unexpected error occurred on the server.

Possible Fix: If the problem persists, contact support with details of the request.

401 Unauthorized

Description: Authentication is required or has failed.

Possible Fix: Ensure you are providing valid authentication credentials.

Example Use Cases

Here are some example use cases for interacting with the Recipe API:

1. Retrieve All Recipes

Request: GET http://your-api-url.com/api/recipes

Response: Returns a list of recipes with their details.

2. Add a New Recipe

Request: POST http://your-api-url.com/api/recipes

Request Body:

{
  "name": "Spaghetti Carbonara",
  "ingredients": ["spaghetti", "eggs", "pancetta", "cheese"],
  "instructions": "Cook spaghetti, mix with eggs and cheese, add pancetta.",
  "img": "URL_to_image"
}

Response: Returns the newly added recipe.

3. Update an Existing Recipe

Request: PUT http://your-api-url.com/api/recipes/{id}

Request Body:

{
  "name": "Updated Spaghetti Carbonara",
  "ingredients": ["spaghetti", "eggs", "pancetta", "cheese", "black pepper"],
  "instructions": "Cook spaghetti, mix with eggs and cheese, add pancetta and pepper.",
  "img": "URL_to_image"
}

Response: Returns the updated recipe.

4. Delete a Recipe

Request: DELETE http://your-api-url.com/api/recipes/delete/{id}

Response: Returns a confirmation of deletion.
